# blackjack-music-sync

**Keep your local music library in sync with YouTube playlists - automatically, safely, and without ever touching files you already have.**

Built for people who curate YouTube playlists as their music library and want those playlists mirrored locally as MP3s - on a NAS, a home server, or any network drive - with zero manual work after setup.

---

## How it works

Every time you run the script, for each playlist it:

1. Fetches the full track list and metadata from YouTube (no download yet)
2. Reads what you already have locally
3. Matches existing files to YouTube tracks using **duration + title** - no re-downloads, no duplicates
4. Records matched IDs in `_id_map.txt` (the permanent source of truth per folder)
5. Downloads only the tracks that are genuinely missing, as high-quality MP3
6. Updates the `.m3u` playlist file so your media player stays in sync

It **never deletes** anything. It **never overwrites** anything. Existing files are untouched.

## Requirements

- Windows with PowerShell 7+
- [yt-dlp](https://github.com/yt-dlp/yt-dlp) - `winget install yt-dlp`
- [ffmpeg](https://ffmpeg.org) (includes ffprobe) - `winget install ffmpeg`
- A destination folder - local, NAS, or network drive

## Setup

**1. Clone the repo**
```powershell
git clone https://github.com/0xDonnie/blackjack-music-sync
cd blackjack-music-sync
```

**2. Create your config**
```powershell
Copy-Item config.example.ps1 config.ps1
```
Then open `config.ps1` and fill in your folder path and playlist URLs.

**3. Launch the GUI**

Double-click **`blackjack-music-sync.cmd`** and the GUI opens. You'll see a brief flash of a console window, then the dark-theme GUI appears.

Alternative from terminal:
```powershell
pwsh -NoProfile -ExecutionPolicy Bypass -File gui_v1.ps1
```

For headless / automated use without the GUI:
```powershell
pwsh -NoProfile -ExecutionPolicy Bypass -File sync_playlists_v1.ps1
```

## File structure

Each playlist folder ends up looking like this:

```
Music\
  My Chill Playlist\
    01. Song title.mp3
    02. Another song.mp3
    ...
    My Chill Playlist.m3u     # auto-updated playlist for media players
    _id_map.txt               # maps filename => YouTube ID (source of truth)
    .archive                  # used by yt-dlp to skip already-downloaded tracks
```

---

## Adding a new playlist

Open `config.ps1` and add a line to `$PLAYLISTS`:

```powershell
"My New Playlist" = "https://www.youtube.com/playlist?list=..."
```

The folder name becomes the subfolder under your base directory. Run the script and it handles the rest.

## Why duration + title matching?

Most tools either re-download everything or rely on the YouTube ID being embedded in the filename. This script takes a different approach: it uses `ffprobe` to read the actual duration of each local MP3 and compares it against the YouTube track duration (within a 5-second tolerance), combined with a title similarity check. This lets it recognize files you already have even if they were downloaded with a different tool, renamed, or never had an ID in their name.

## Automation

Use the **Monitor tab** in the GUI to schedule automatic update checks (Daily / Every 3 days / Weekly / Monthly / Semestral) — it registers a Windows Scheduled Task for you. When new tracks are detected, you can click "Sync pending now" from the same tab.

For fully unattended sync via Task Scheduler manually:
```
pwsh.exe -NoProfile -ExecutionPolicy Bypass -File "C:\path\to\sync_playlists_v1.ps1"
```

## Scope

Out of the box, this tool syncs **public and unlisted** YouTube playlists. That covers the vast majority of curated music playlists.

It does **not** out-of-the-box support:
- **Private playlists or videos** that require you to be signed in (the playlist owner has not made them public/unlisted)
- **Members-only / Premium-only** content
- **Age-restricted videos** that require an account to confirm age

These cases require yt-dlp to authenticate as your YouTube account by reading cookies from your browser. It's possible but it's an advanced setup with caveats - see [COOKIES.md](COOKIES.md) if you really need it.

## Notes

- **YouTube Mixes/Radio don't work.** URLs with `list=RD...` are auto-generated by YouTube and change every time you open them - they can't be synced. Only regular playlists (`list=PL...`) are supported. If you like songs from a Mix, create a new playlist on YouTube, add those songs to it, and use that playlist URL instead.
- Some public videos may still fail with "This video is not available". This usually means yt-dlp couldn't solve the JS signature challenge YouTube imposes. The sync script enables `--remote-components ejs:github` automatically to fix this for most cases.
- Some playlist tracks become permanently unavailable on YouTube (uploader-deleted, copyright-removed, region-blocked, etc). The script tracks these in `_unavailable.txt` per folder and the GUI counts them separately so they don't show up as "missing" forever.
- yt-dlp updates frequently. Update with `python -m pip install --upgrade yt-dlp` (pip install) or `winget upgrade yt-dlp` (winget install) if you start hitting extraction errors.
- All activity is logged to `sync.log` in your base folder.
